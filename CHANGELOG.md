# Changelog

All notable changes to **homepage-diagnostic** are documented here.

## [1.1.0] — 2026-07-06

Richer data reach, an honest multi-axis profile, and shareable output.

### Added
- **Full-page link crawl** (not just `<head>`/sitemap): `collector/collect.py`-adjacent
  harvest classifies every outbound link (press, place, social, delivery, map). This
  surfaced press/certification assets a head-only parse missed.
- **Off-site probes** (`collector/offsite.py`): Naver blog volume, Naver Place review
  count, **당근 local-profile** (단골·후기·별점 via web-search discovery), Threads
  (gated → 미상), keyword market (declared + Naver autocomplete). Each read = fact +
  citation; each gated value = 미상 + verify URL.
- **Search Console connector** (`collector/searchconsole.py`): owner-gated GSC search
  analytics (query · impressions · clicks · CTR · position, plus country/device/page).
  Token stays local (env var). Naver Search Advisor via CSV export. Upgrades keyword
  volume/rank from 미상 to fact.
- **Self-contained output** (`collector/selfcontain.py`): inlines images (base64) and
  drops CDN font/CSS deps so a report opens offline, prints to PDF, and passes strict CSP.
- **Multi-axis profile (radar)** replaces the single composite score: 발견성 · 신뢰·평판 ·
  재방문·오운드 · 정보 위생 axes, each with its own confidence grade; composite shown small;
  unmeasured axes (성과·수익) left unscored rather than invented.
- **Authenticated-connector + data-source docs** (`references/data-sources.md`): what each
  insight source yields, its reach technique, and its default grade.
- A neutral committed example (`examples/output/obsidian-md.*`). Full client
  reports are generated by running the skill and are kept out of the repo.

### Changed
- `report-template.html` upgraded to the bright/editorial v4+ look with a radar profile,
  a GSC section slot, big numbers, funnel + charts. Old dark template retired.
- Fetch layer delegates hard fetches to the [insane-search](https://github.com/fivetaku/insane-search)
  engine when present (imported, not vendored); falls back to stdlib.

## [1.0.0] — 2026-07-06

First public release: packaged as a Claude Code plugin + portable skill.

### Added
- **Skill** (`skills/homepage-diagnostic/SKILL.md`): 7-step pipeline —
  fetch homepage `<head>` → seed research from JSON-LD → **research public
  off-site sources with numbered citations** → read crawl → diagnose (levels
  scorecard, core-question, channel-fit) → priorities + "지금 아님" →
  verification map + references.
- **Evidence grading** (`references/evidence-grading.md`): 사실 / 추정 /
  데이터 없음, plus off-site citation rules (read-and-cite = fact, unread
  number = no-data, tone-is-fact-quality-is-not, no orphan citations).
- **Collector contract** (`references/meta-schema.json`): adds
  `business_info` research seeds (hours, price, geo, reservations, `same_as`,
  `has_map`).
- **Report template** (`assets/report-template.html`): 8-section cited
  local-marketing report — legend, cited facts, levels scorecard, optional
  core-question Q&A, channel-fit table, ranked priorities, verification map,
  references.
- **Reference collector** (`collector/collect.py`, stdlib only) emitting
  schema-conformant JSON + a `--score` metrics summary.
- **Test harness** (`tests/run.sh`): prints comparable result values
  (meta fields, seo flags, JSON-LD, business_info, research seeds,
  gradeable_facts) per URL.
- **verify.sh**: structural integrity + token-contract + collector checks.
- Neutral committed example (`examples/output/obsidian-md.draft.html`); real client
  reports are kept out of the repo (owner-gated data).
